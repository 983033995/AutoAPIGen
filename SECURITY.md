## 🚨 报告安全漏洞

我们非常重视 AutoAPIGen 的安全性。如果您发现了安全漏洞，请负责任地向我们报告。

### 🔒 私密报告

**请不要在公开的 GitHub Issues 中报告安全漏洞。**

请通过以下方式私密报告安全问题：

1. **GitHub 安全报告**（推荐）
   - 访问 [GitHub 安全报告页面](https://github.com/983033995/AutoAPIGen/security/advisories/new)
   - 填写详细的漏洞信息
   - 我们会在 24 小时内回复

2. **邮件报告**
   - 发送邮件至：[security@example.com]（请替换为实际邮箱）
   - 邮件主题：`[SECURITY] AutoAPIGen 安全漏洞报告`
   - 请使用 PGP 加密敏感信息（公钥见下方）

### 📋 报告内容

请在安全报告中包含以下信息：

#### 基本信息
- **漏洞类型**：例如 XSS、代码注入、权限提升等
- **影响范围**：受影响的版本和功能
- **严重程度**：您对漏洞严重程度的评估

#### 详细描述
- **漏洞描述**：详细说明安全问题
- **攻击向量**：如何利用这个漏洞
- **影响评估**：可能造成的损害
- **复现步骤**：详细的复现步骤

#### 技术信息
- **环境信息**：VSCode 版本、操作系统等
- **代码位置**：相关的文件和行号
- **概念验证**：如果可能，提供 PoC 代码
- **修复建议**：如果有修复建议，请提供

### ⏰ 响应时间

我们承诺：

- **24 小时内**：确认收到报告
- **72 小时内**：初步评估和分类
- **7 天内**：详细分析和修复计划
- **30 天内**：发布安全修复（根据严重程度调整）

## 🔐 安全最佳实践

### 👥 用户安全建议

作为 AutoAPIGen 用户，请遵循以下安全最佳实践：

#### 插件使用
- ✅ 只从官方 VSCode 市场安装 AutoAPIGen
- ✅ 定期更新到最新版本
- ✅ 仔细审查生成的代码
- ❌ 不要在生成的代码中包含敏感信息

#### 配置安全
- ✅ 使用环境变量存储敏感配置
- ✅ 定期轮换 API 密钥和令牌
- ✅ 限制 API 访问权限
- ❌ 不要在配置文件中硬编码密钥

#### 代码安全
- ✅ 对生成的代码进行安全审查
- ✅ 使用 HTTPS 进行 API 通信
- ✅ 实施适当的输入验证
- ❌ 不要信任未验证的外部数据

### 🛠️ 开发者安全指南

如果您是 AutoAPIGen 的贡献者：

#### 代码安全
- 遵循安全编码规范
- 进行代码安全审查
- 使用静态安全分析工具
- 实施适当的错误处理

#### 依赖管理
- 定期更新依赖项
- 使用 `npm audit` 检查漏洞
- 避免使用有已知漏洞的包
- 最小化依赖项数量

#### 数据处理
- 验证所有输入数据
- 使用参数化查询
- 实施适当的访问控制
- 加密敏感数据

## ⚠️ 已知安全考虑

### 代码生成安全

- **代码注入风险**：生成的代码可能包含恶意内容
  - 缓解措施：代码生成前进行输入验证和清理
  - 建议：用户应审查所有生成的代码

- **API 密钥暴露**：配置中可能包含敏感信息
  - 缓解措施：使用环境变量和安全存储
  - 建议：定期轮换密钥

### 网络安全

- **中间人攻击**：HTTP 通信可能被拦截
  - 缓解措施：强制使用 HTTPS
  - 建议：验证 SSL 证书

- **数据泄露**：API 响应可能包含敏感数据
  - 缓解措施：实施数据过滤和清理
  - 建议：最小化数据传输

## 🚨 安全事件响应

### 事件分类

我们使用以下严重程度分类：

#### 🔴 严重 (Critical)
- 远程代码执行
- 权限提升
- 数据泄露
- 系统完全妥协

#### 🟠 高 (High)
- 本地代码执行
- 敏感信息泄露
- 认证绕过
- 重要功能破坏

#### 🟡 中 (Medium)
- 拒绝服务
- 信息泄露
- 权限绕过
- 功能异常

#### 🟢 低 (Low)
- 信息收集
- 配置问题
- 轻微功能影响

### 响应流程

1. **接收报告**：确认收到安全报告
2. **初步评估**：评估漏洞的真实性和严重程度
3. **深入分析**：详细分析漏洞的影响范围
4. **开发修复**：开发和测试安全修复
5. **发布修复**：发布安全更新
6. **公开披露**：在适当时间公开漏洞信息

### 修复时间表

- **严重漏洞**：24-48 小时内修复
- **高危漏洞**：3-7 天内修复
- **中危漏洞**：1-2 周内修复
- **低危漏洞**：下个版本修复
