# Changelog

## [1.3.1] - 2026-04-16

### 🐛 修复

- **修复主面板切换项目时污染 `.vscode/autoApiGen.json` 的问题**：切换项目时仅持久化真实配置字段，不再把 `apiDetailList`、`apiTreeList`、`apiProjectList`、`apiDataSchemas` 等运行态缓存写入本地配置文件；保存配置时也会清理历史残留的无用字段，避免配置文件膨胀到数万行。
- **修复插件初始化异常时界面卡死在 loading 的问题**：当接口获取或工作区状态初始化失败时，主面板会正确退出 loading，并展示错误态与 **重新加载** 按钮，无需重启扩展或 VS Code。
- **修复初始化后续状态链路的异常恢复能力**：初始化后的消息处理增加更稳妥的错误透传与状态收敛，避免因单次异常导致侧边栏面板长期不可恢复。

## [1.3.0] - 2026-04-12

### 🚀 新增

- **新增 `aag` CLI 工具**：提供 `init`、`query`、`groups`、`generate`、`ui` 五类命令，支持脱离 VS Code 的接口查询与代码生成工作流。
- **新增交互式树钻取模式**：`aag ui` 或 `aag generate` 无参数时可直接在终端浏览分组、搜索接口、批量勾选并生成代码。
- **新增 `aag groups` 命令**：支持输出分组树与接口 ID，便于 AI 和开发者按分组批量生成。
- **新增 AI 友好的 `query --json` 摘要结构**：返回函数名、路径、路径参数、Query、Body、`response200`，降低 AI 二次解析成本。
- **新增 AI Skill 文档源**：补充 `skills/auto-api-gen/SKILL.md`，统一描述“先查本地，再查 CLI，再生成”的 AI 工作流。
- **插件面板新增快捷入口**：底部新增 **AI 支持** 与 **使用文档** 按钮，便于直接跳转到文档入口。

### 🔄 变更

- **AI 支持入口简化**：`AutoAPIGen.enableAISupport` 从直接处理注入逻辑调整为打开文档页，由文档统一承接 CLI 与 AI 集成说明。
- **CLI 输出路径与插件完全对齐**：统一采用 `<config.path>/<appName>/[<projectName(PascalCase)>/]<group>/` 规则，兼容历史项目里以 `/src/services`、`/apps/...` 开头的配置习惯。
- **Monorepo 架构落地**：新增 `packages/core` 与 `packages/cli`，将 HTTP / 类型 / 代码生成公共逻辑抽离，方便 CLI 与插件复用。
- **CLI 生成流程与插件路径逻辑对齐**：项目名插入、分组目录转拼音、根目录拼接方式均与插件保持一致。
- **配置页文案与入口优化**：空状态操作从“刷新”改为 **AI 支持**，同时保留文档快捷入口，降低首次接入成本。

### 🐛 修复

- **修复 `config.path` 解析问题**：CLI 对历史配置中带前导 `/` 的路径不再误判，生成目录与插件行为一致。
- **修复分组退出逻辑**：交互式浏览模式支持在任意层级快速返回与确认退出。
- **修复路径名清洗问题**：分组目录在转拼音前会先移除非法字符，避免生成无效路径。
- **修复英文目录处理问题**：仅中文会被转拼音，英文目录名保持原样，避免过度转换。
- **修复 `query --json` 中 Schema 引用展开问题**：支持 body / response 对 `$ref` 与 `x-apifox-refs` 的解析，摘要信息更完整。
- **修复生成摘要可读性问题**：AI 查询输出不再依赖冗长原始详情结构，改为更稳定的结构化摘要。

## [1.2.13] - 2026-01-29

### 🐛 问题修复

- **Array 类型代码生成修复**：修复 array 类型生成时出现双重 `[]` 的问题（如 `xxxItem[][]`），现在正确生成为 `xxxItem[]` ✅
- **空对象检查逻辑优化**：修复 `buildChildrenOutput` 函数中对 array 类型的空对象检查逻辑，正确检查 `items.properties` 而不是 `noRef.properties` ✅
- **接口定义分号补全**：在 `export type xxx = object` 和 `export type xxx = xxxItem[]` 语句末尾添加分号，确保代码格式正确 ✅
- **更新已存在文件时的代码拼接修复**：修复更新已存在接口文件时，子类型定义被错误拼接到其他接口定义后面的问题 ✅
- **类型定义删除优化**：增强 `removeTypeDefinitions` 函数，支持删除子类型定义（如 `xxxResItem`、`xxxResItemItem`）和独立的类型引用行 ✅

### 🔧 功能增强

- **Array 类型 Interface 生成**：对于 array 类型，现在使用 `items` 对象生成正确的 interface 定义，而不是使用整个 array 对象 ✅
- **子类型命名优化**：`output` 函数调用时传入正确的接口名称（`childrenInterfaceName` 而不是 `childrenFaceName`），确保生成的代码更加规范 ✅

## [1.2.12] - 2025-01-25

### 🔧 功能增强

- **接口字段描述支持**：生成 interface 时支持 `description` 字段作为注释，优先使用 `title`，没有则使用 `description` ✅
- **x-apifox-refs 引用解析**：完善 `transformSchema` 函数，支持解析 `x-apifox-refs` 中的 `$ref` 引用并合并到 properties ✅
- **x-apifox-overrides 覆盖配置**：支持通过 `x-apifox-overrides` 覆盖引用 schema 中的字段定义 ✅

### 🐛 问题修复

- **单行 type 定义删除修复**：修复 `removeTypeDefinitions` 函数处理 `export type xxx = boolean` 单行类型定义时，导致后续内容被误删的问题 ✅
- **空对象 interface 警告修复**：当 body 参数为空或 properties 为空时，生成 `export type xxx = object` 替代空 interface，避免 ESLint 警告 ✅
- **空引用对象处理优化**：当 `$ref` 引用的 schema 是空对象时，直接返回 `{ [key: string]: any }` 而不是生成空 interface ✅
